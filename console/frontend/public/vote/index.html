按环节编辑（与 vote-config 字段一致）

结构与 vote-config.js 相同：初赛用 round1PkByRoundId 式两人（左/右），复活 / 决赛用 candidates 式列表。默认从 vote-config 载入；展开某一环节改标题、副标题与选手后一次发布写入 Firestore voteUi.rounds.{roundId}。观众 vote.html?roundId=… 只读当前环节块。

初赛每环节须 2 人；其它环节 1～6 人。发布时 Cloud Function 校验 s1～s10、行号等。若某环节留空选手，观众端该轮会回退到 vote-config（初赛五组仍优先 round1PkByRoundId）。

部署：firebase deploy --only firestore:rules,functions 后设置 Secret STAFF_PUBLISH_SECRET。详见 console/docs/README-vote-firebase-static.md。
